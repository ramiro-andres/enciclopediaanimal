# Internacionalización (i18n) — Atlas Animal

**Historia:** US-UX-07  
**Fase 1 (actual):** Cromo de interfaz ES/EN (botones, labels, títulos de sección, placeholders, empty states, disclaimers de UI)  
**Fase 2 (futuro):** Traducción de fichas clínicas y glosario

## Estrategia

| Capa | Enfoque |
|------|---------|
| Cromo UI (nav, herramientas, labels de ficha, tour, tab bar…) | Claves en `js/i18n.js` + `data-i18n` / `I18n.t()` |
| Contenido clínico (`enciclopedia.json`, nombres de razas/enfermedades, cuerpos de protocolo) | **Español único** por ahora — volumen y revisión veterinaria |
| Glosario médico (términos y definiciones) | Español — equivalencias EN en fase 2 |

## Uso

```javascript
I18n.setLang('en');  // persiste en localStorage (atlas_lang)
I18n.t('nav.home');  // "Home"
I18n.apply();        // aplica data-i18n / data-i18n-html / data-i18n-placeholder / data-i18n-aria
```

Al cambiar idioma se dispara `atlas:langchange` y `app.js` re-renderiza vistas dinámicas (welcome stats, tools, fichas abiertas, etc.).

## Limitación conocida (contenido clínico)

Al elegir inglés, el **cromo de la UI** se traduce; **no** se traduce masivamente el contenido clínico (nombres de razas, enfermedades, textos de protocolos, entradas del glosario procedentes de datos). Eso es intencional en fase 1.

## Decisiones

**JSON paralelo** (`enciclopedia.en.json`) descartado en fase 1 por duplicar ~2 MB y 2 500+ enfermedades.  
**Recomendación fase 2:** campos opcionales `_en` por raza o pipeline de traducción asistida con revisión humana obligatoria.

Alias tipográfico mantenido: `tab.glosary` → mismo valor que `tab.glossary`.
